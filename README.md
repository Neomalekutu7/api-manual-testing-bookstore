BookStore API Manual Testing

Project Overview

This project demonstrates manual testing of a RESTful BookStore API. 
It includes:
Requirements Traceability Matrix (RTM)
Test Plan
20 Manual Test Cases
Execution Report
Test Summary
AI-assisted documentation and test case generation

Tools Used:

Postman
Google Sheets
Google Docs
AI tools (for generating test cases and summarizing reports)

Key Learnings

Planning and executing API test cases manually
Validating API response structure and status codes
Testing authentication and error handling
Leveraging AI tools to improve efficiency in test case creation and reporting

Instructions for Testing

Import the provided API endpoints into Postman.
Execute each test case manually.
Record the response body and status code for each request in the execution report.
Log any bugs encountered during testing in the bug report.
Summarize overall findings and observations in the test summary report.
Use AI tools to assist in analyzing responses, generating summaries, or creating additional test scenarios.
